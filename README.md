# Digital Payments Fraud Analytics

## Project Overview

This project analyzes digital payment transactions to identify fraud patterns, transaction trends, customer behavior, and operational risks. The analysis combines SQL, Python, and Power BI to transform raw transaction data into actionable business insights.

---

## Tools & Technologies

- SQL (MySQL)
- Python
- Pandas
- NumPy
- Matplotlib
- Jupyter Notebook
- Power BI

---

## Dataset

The dataset contains digital payment transactions with attributes such as:

- Transaction Type
- Transaction Amount
- Merchant Category
- Device Type
- Sender & Receiver Information
- Transaction Status
- Fraud Flag
- Time & Date Features

---

## SQL Analysis

Key business questions answered:

1. Total transactions and fraud rate
2. Fraud rate by transaction type
3. Fraud rate by device type
4. Transaction volume by merchant category
5. Transaction volume by hour of day
6. Weekend vs Weekday transaction patterns
7. Sender age group analysis
8. State-wise transaction distribution

---

## Python Analysis

Performed exploratory data analysis using:

- Data Cleaning
- Feature Understanding
- Fraud Trend Analysis
- Hourly Fraud Analysis
- Merchant Category Analysis
- Bank-wise Fraud Analysis
- Age Group Analysis

## Power BI Dashboard

Dashboard includes:

- KPI Cards
- Fraud Rate Tracking
- Transaction Distribution
- Merchant Category Analysis
- Device Analysis
- Time-based Trends
- Interactive Filters

---

## Key Insights

- Transport and Education categories showed relatively higher fraud rates.
- Fraud activity varied significantly across transaction hours.
- Younger and older customer segments showed elevated fraud percentages.
- Certain banks displayed higher fraud incidence than others.
- Transaction type and device usage impacted fraud occurrence.
